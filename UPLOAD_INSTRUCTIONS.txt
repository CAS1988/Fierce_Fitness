TIGER v3.5.4 — FRESH GITHUB REPOSITORY UPLOAD

IMPORTANT
Create a brand-new repository with a NEW repository name, for example:
  tiger-training-fresh

Using a new name creates a new GitHub Pages URL and completely avoids the old
phone application's service-worker scope. Reusing the deleted repository's exact
name can reuse the same URL and allow the old installed app to remain involved.

UPLOAD
1. Extract this ZIP on your computer.
2. Create a new GitHub repository. A public repository is simplest for Pages.
3. Upload ALL extracted contents to the repository root.
4. Do not upload the ZIP itself.
5. Confirm the repository root directly shows:
     index.html
     app.js
     styles.css
     sw.js
     manifest.webmanifest
     version.txt
     .github
6. Commit the upload to the main branch.
7. Open repository Settings > Pages.
8. Under Build and deployment, select GitHub Actions as the source.
9. Open the Actions tab and wait for "Deploy Tiger to GitHub Pages" to show a green tick.

VERIFY BEFORE INSTALLING
1. Open:
     https://YOUR-USERNAME.github.io/NEW-REPOSITORY-NAME/version.txt
2. It must say "Tiger v3.5.4".
3. Then open the normal Pages address:
     https://YOUR-USERNAME.github.io/NEW-REPOSITORY-NAME/
4. In Tiger, open Settings and confirm "Tiger v3.5.4" at the bottom.

PHONE
Remove the old Tiger home-screen shortcut before installing the new Pages URL.
Then open the new URL in Chrome (Android) or Safari (iPhone) and add/install it
to the home screen.

DATA
The cache reset does not delete Tiger logs or settings. When the old and new
repositories use the same GitHub username (the same username.github.io origin),
Tiger should continue to find the existing v3.5.2-format local data. Exporting a
backup first is still sensible if the old app remains accessible. Data will not
carry automatically if the new site uses a different GitHub username or domain.
