TIGER v3.5.6 — FAST-START GITHUB PAGES UPDATE

UPLOAD TO YOUR EXISTING REPOSITORY
1. Extract this ZIP on your computer.
2. In the GitHub repository currently hosting Tiger, upload ALL extracted files.
3. Replace files with the same names when GitHub asks.
4. Do not upload the ZIP itself or place the extracted files inside another folder.
5. Confirm the repository root directly shows index.html, app.js, styles.css,
   sw.js, manifest.webmanifest and version.txt.
6. Commit the upload to the main branch.

PAGES SETTING
Use the same publishing method that is already working:
  Settings > Pages
  Source: Deploy from a branch
  Branch: main
  Folder: /(root)

VERIFY
1. Wait for the Pages deployment to complete.
2. Open:
     https://YOUR-USERNAME.github.io/YOUR-REPOSITORY-NAME/version.txt
3. Confirm it says "Tiger v3.5.6".
4. Fully close the installed Tiger app and reopen it. The first opening after the
   update may fetch the new files; following launches should be substantially faster.
5. In Tiger, open Settings and confirm "Tiger v3.5.6" at the bottom.

DATA

history, settings and progression records remain on the same phone/browser origin.
